# Claude Artifact Architecture

**Version:** 2.0  
**Last Updated:** 2026-04-14  
**Status:** Production

---

## Overview

The Claude artifact generation system creates a modular, lazy-loading directory structure optimized for large language model consumption. Unlike the old JSON-based system, the new Markdown-based architecture reduces initial token load by 97% through intelligent lazy loading.

## Component Architecture

### 1. CLAUDE.md (Routing File)

**Purpose:** Entry point for Claude to determine which agent to use

**Generated By:** `claude_md.py::generate_claude_md()`

**Template:** `templates/claude/CLAUDE.md.j2`

**Contains:**
- Agent registry table (all available agents)
- Routing rules (keyword matching)
- Core instructions (load sequence)

**Size:** ~3.7KB

**Frequency:** Generated once per build

---

### 2. Agent Files

**Purpose:** Define primary agents that users can select

**Generated By:** `claude_builder.py::_render_agent_file()`

**Template:** `templates/claude/agent.md.j2`

**Contains:**
- Agent purpose and when to use
- Role description (system prompt)
- Workflow reference
- Subagents table
- Skills table
- Startup sequence instructions

**Size:** ~2KB each

**File Naming:** `{agent-name}-agent.md` (kebab-case)

**Examples:**
- `code-agent.md`
- `debug-agent.md`
- `architect-agent.md`

---

### 3. Subagent Files

**Purpose:** Specialized helpers for focused tasks

**Generated By:** `claude_builder.py::_render_subagent_file()`

**Template:** `templates/claude/subagent.md.j2` (fallback)

**Source:** `agents/{agent}/subagents/{subagent}/{variant}/prompt.md`

**Contains:**
- Subagent purpose
- Delegation context (which agent calls this)
- Focused instructions
- Return to parent instructions

**Size:** ~1KB each

**File Naming:** `{subagent-name}.md` (kebab-case)

**Examples:**
- `code-reviewer.md` (from code/subagents/code-reviewer)
- `rubber-duck.md` (from debug/subagents/rubber-duck)
- `accessibility-checker.md` (from review/subagents/accessibility)

---

### 4. Workflow Files

**Purpose:** Multi-step processes that agents follow

**Generated By:** `claude_builder.py::_render_workflow_file()`

**Template:** `templates/claude/workflow.md.j2` (fallback)

**Source:** `workflows/{workflow}/{variant}/workflow.md`

**Contains:**
- Workflow overview
- Language detection step
- Sequential steps with actions
- Resource loading instructions
- Completion criteria

**Size:** ~3KB each

**File Naming:** `{workflow-name}.md` (kebab-case)

**Examples:**
- `feature-implementation.md`
- `code-review.md`
- `deep-research.md`

---

### 5. Convention Files

**Purpose:** Coding standards and rules

**Generated By:** `convention_generator.py::generate_all_conventions()`

**Source:** `agents/core/system.md`, `conventions.md`, `session.md`, `conventions-{language}.md`

**Types:**

**5.1 Core Convention (general.md)**
- Combines: system.md + conventions.md + session.md
- Size: ~10KB
- Location: `.claude/conventions/core/general.md`
- Always loaded first

**5.2 Language Conventions ({language}.md)**
- One file per language (30 total)
- Size: ~8KB each
- Location: `.claude/conventions/languages/{language}.md`
- Loaded by workflow when language detected

---

### 6. Skills

**Purpose:** Reusable capabilities

**Generated By:** Existing skill system (not changed)

**Source:** `skills/{skill}/SKILL.md`

**Location:** `.claude/skills/{skill}/SKILL.md`

**Examples:**
- `feature-planning/SKILL.md`
- `post-implementation-checklist/SKILL.md`

## Data Flow

### Build Process

```
1. User runs: promptosaurus init (tool=claude)
   │
   ▼
2. PromptBuilder.build()
   │
   ├─▶ For each agent:
   │   │
   │   ├─▶ ClaudeBuilder.build(agent)
   │   │   │
   │   │   ├─▶ _render_agent_file()
   │   │   │   └─▶ Uses agent.md.j2 template
   │   │   │
   │   │   ├─▶ _render_subagent_file() (for each subagent)
   │   │   │   ├─▶ Load from agents/{agent}/subagents/{subagent}/{variant}/prompt.md
   │   │   │   └─▶ Fallback to subagent.md.j2 template
   │   │   │
   │   │   └─▶ _render_workflow_file() (for each workflow)
   │   │       ├─▶ Load from workflows/{workflow}/{variant}/workflow.md
   │   │       └─▶ Fallback to workflow.md.j2 template
   │   │
   │   └─▶ Returns dict[str, str] (file paths → content)
   │
   ├─▶ generate_claude_md(agents)
   │   └─▶ Uses CLAUDE.md.j2 template
   │
   ├─▶ generate_all_conventions()
   │   ├─▶ generate_core_convention()
   │   │   └─▶ Combines system + conventions + session
   │   └─▶ For each language:
   │       └─▶ generate_language_convention(language)
   │
   └─▶ _write_output()
       └─▶ Writes all files to .claude/ directory
```

### Runtime Loading (Claude's Perspective)

```
1. Claude receives request
   │
   ▼
2. Load CLAUDE.md (3.7KB)
   │
   ▼
3. Match request to agent via routing rules
   │
   ▼
4. Load .claude/conventions/core/general.md (10KB)
   │
   ▼
5. Load .claude/agents/{selected-agent}.md (2KB)
   │
   ▼
6. Agent file instructs: Load workflow
   │
   ▼
7. Load .claude/workflows/{workflow}.md (3KB)
   │
   ▼
8. Workflow instructs: Detect language
   │
   ▼
9. Load .claude/conventions/languages/{language}.md (8KB)
   │
   ▼
10. Workflow instructs: Load subagent (if needed)
    │
    ▼
11. Load .claude/subagents/{subagent}.md (1KB)
    │
    ▼
12. Workflow instructs: Load skill (if needed)
    │
    ▼
13. Load .claude/skills/{skill}/SKILL.md (2KB)

Total: ~30KB loaded dynamically vs 728KB loaded upfront
```

## Naming Conventions

### File Naming Rules

**Agents:** `{name}-agent.md`
- Examples: `code-agent.md`, `debug-agent.md`
- Function: `agent_to_file_name()`

**Subagents:** `{name}.md`
- Examples: `code-reviewer.md`, `rubber-duck.md`
- Function: `subagent_to_file_name()`
- Special: Handles paths like "debug/rubber-duck" → "rubber-duck.md"

**Workflows:** `{name}.md`
- Examples: `feature-implementation.md`, `code-review.md`
- Function: `workflow_to_file_name()`

**Skills:** `{name}/SKILL.md`
- Examples: `feature-planning/SKILL.md`
- Function: `skill_to_directory_name()`

**Conventions:** `{type}/{name}.md`
- Examples: `core/general.md`, `languages/python.md`

**All use kebab-case** (lowercase with hyphens)

## Template System

### Jinja2 Templates

**Location:** `promptosaurus/templates/claude/`

**Templates:**
1. `agent.md.j2` - Agent files
2. `CLAUDE.md.j2` - Routing file
3. `subagent.md.j2` - Subagent files
4. `workflow.md.j2` - Workflow files

### Template Variables

**agent.md.j2:**
```jinja
- agent.name
- agent.description
- agent.system_prompt
- when_to_use
- workflow
- workflow_steps
- subagents (list of dicts)
- skills (list of dicts)
- notes
```

**CLAUDE.md.j2:**
```jinja
- last_updated
- agent_count
- persona_name
- agents (list of dicts)
- routing_categories (list of dicts)
```

**subagent.md.j2:**
```jinja
- subagent.name
- subagent.description
- subagent.instructions
- parent_agent
- notes
```

**workflow.md.j2:**
```jinja
- workflow.name
- workflow.description
- workflow.duration
- workflow.prerequisites
- workflow.overview
- workflow.steps (list of dicts)
- workflow.completion_criteria
- notes
```

## Fallback Strategy

The system uses a three-tier fallback strategy for content loading:

### For Workflows:
1. **Try primary variant:** `workflows/{workflow}/{variant}/workflow.md`
2. **Try other variant:** `workflows/{workflow}/{other_variant}/workflow.md`
3. **Use template:** `workflow.md.j2` with placeholder content

### For Subagents:
1. **Try primary variant:** `agents/{agent}/subagents/{subagent}/{variant}/prompt.md`
2. **Try other variant:** `agents/{agent}/subagents/{subagent}/{other_variant}/prompt.md`
3. **Use template:** `subagent.md.j2` with placeholder content

This ensures builds never fail due to missing content.

## Extension Points

### Adding New Languages

1. Add `conventions-{language}.md` to `promptosaurus/agents/core/`
2. Rebuild: `promptosaurus switch --tool claude`
3. New file appears in `.claude/conventions/languages/{language}.md`

### Adding Custom Agents

1. Add agent directory to `promptosaurus/agents/{agent}/`
2. Create `prompt.md` with agent definition
3. Optionally add subagents in `subagents/` subdirectory
4. Rebuild: `promptosaurus switch --tool claude`

### Adding Custom Workflows

1. Add workflow directory to `promptosaurus/workflows/{workflow}/`
2. Create `minimal/workflow.md` and/or `verbose/workflow.md`
3. Reference from agent configuration
4. Rebuild: `promptosaurus switch --tool claude`

## Security Considerations

### Content Sanitization

- All Markdown content is generated from trusted sources (bundled files)
- No user input is directly inserted into templates
- File paths are validated via naming_utils functions

### File System Access

- All writes confined to output directory
- Automatic directory creation uses `mkdir(parents=True, exist_ok=True)`
- No symlinks created or followed

### Version Control

- All generated files are project artifacts (should be committed)
- No sensitive data included in generated files
- Convention files come from bundled sources only

## Troubleshooting

### Missing Subagent Content

**Symptom:** Subagent file contains placeholder text instead of real content

**Cause:** Subagent source file not found in expected location

**Solution:** 
1. Check if subagent exists in `agents/{agent}/subagents/{subagent}/`
2. Verify variant (minimal/verbose) is correct
3. Check naming (subagent paths use kebab-case)

### Workflow Not Loading

**Symptom:** Workflow file contains placeholder steps

**Cause:** Workflow source file not found

**Solution:**
1. Check if workflow exists in `workflows/{workflow}/{variant}/`
2. Verify workflow name matches (use kebab-case)
3. Check if at least one variant (minimal or verbose) exists

### Build Failures

**Symptom:** Build process fails with errors

**Cause:** Usually validation errors or missing required fields

**Solution:**
1. Check agent IR models have required fields (name, description, system_prompt)
2. Verify template files exist in `templates/claude/`
3. Check Jinja2 syntax in templates

## Migration from Old System

**Automatic Migration:**

The artifact system automatically handles migration:

1. When switching to Claude, `custom_instructions/` is removed (in artifacts.py remove list)
2. `.claude/` directory is created fresh
3. CLAUDE.md is generated
4. All convention files are generated from bundled sources

**No manual migration steps needed.**

**Backward Compatibility:**

The old JSON format is not supported. When you switch to Claude:
- Old `custom_instructions/` is deleted automatically
- New `.claude/` structure is created
- All content comes from bundled sources (no data loss)
