# Requirements Gathering Workflow

## Goal
Systematically collect, validate, and document product requirements from all stakeholders.

## Quick Steps

1. **Identify Stakeholders**
   - Users, customers, internal teams, partners

2. **Gather Requirements**
   - User interviews, surveys, analytics review
   - Competitive analysis, market research

3. **Document Requirements**
   - User stories: "As a [user], I want [goal], so that [reason]"
   - Acceptance criteria using INVEST framework

4. **Validate & Prioritize**
   - Stakeholder review sessions
   - Technical feasibility assessment

5. **Create Requirements Doc**
   - Problem statement, success metrics, constraints

## Key Frameworks

- **INVEST:** Independent, Negotiable, Valuable, Estimable, Small, Testable
- **MoSCoW:** Must have, Should have, Could have, Won't have
- **Jobs to Be Done:** Focus on user outcomes, not features

## Success Factors

- Clear acceptance criteria for each requirement
- Validated by actual users or data
- Technical feasibility confirmed
- Business value quantified
- Dependencies identified

## Template Output

**User Story:** As a [user type], I want [functionality], so that [business value]
**Acceptance Criteria:** Given [context], When [action], Then [outcome]
