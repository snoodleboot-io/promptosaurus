# UX Validation Workflow

## Goal
Validate UX designs with users before investing in development resources.

## Quick Steps

1. **Prepare Prototypes**
   - Low-fi wireframes or high-fi mockups
   - Clickable prototype if possible

2. **Define Test Scenarios**
   - Key user tasks to validate
   - Success criteria for each task

3. **Recruit Test Users**
   - 5-8 target users
   - Mix of new and experienced users

4. **Conduct Testing**
   - Task-based usability testing
   - Think-aloud protocol
   - Record sessions

5. **Analyze Results**
   - Task success rates
   - Common friction points
   - User satisfaction scores

6. **Iterate Design**
   - Fix critical issues
   - Re-test if needed

## Key Frameworks

- **System Usability Scale (SUS):** Standardized satisfaction metric
- **Task Success Rate:** Completion percentage
- **Time on Task:** Efficiency measurement

## Success Factors

- Testing with real target users
- Clear task scenarios defined
- Issues prioritized by severity
- Design iterations based on data
- Stakeholder alignment on changes

## Template Output

**Task:** [User goal]
**Success Rate:** [X%]
**Key Issues:** [Problem 1, Problem 2]
**Recommendation:** [Design change]
