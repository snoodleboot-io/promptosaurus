## Purpose

This workflow guides teams through build responsive user interfaces.

## Key Steps

1. **Understand Requirements** - Gather requirements and constraints
2. **Evaluate Options** - Consider different approaches
3. **Design Solution** - Select and design the solution
4. **Implement** - Execute the implementation
5. **Validate** - Test and validate the solution
6. **Document** - Record decisions and learnings

## Key Concepts

- breakpoints
- mobile-first design
- accessibility
- testing responsiveness

## Success Criteria

- All requirements are met
- Trade-offs are documented
- Team understands the approach
- Solution is maintainable
