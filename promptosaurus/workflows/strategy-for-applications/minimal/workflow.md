# Application Strategy Workflow (Minimal)

## Purpose
Define comprehensive strategy for building applications including architecture, tech stack, scalability, and implementation approach.

## Steps

### 1. Define Business Objectives
- What problem does this application solve?
- Who are the users (internal, external, scale)?
- What are the success criteria (performance, features, timeline)?
- What are the constraints (budget, team size, compliance)?

### 2. Analyze Requirements
- Functional requirements (features, user flows)
- Non-functional requirements (performance, security, reliability)
- Integration requirements (external APIs, services, data sources)
- Compliance requirements (GDPR, HIPAA, SOC 2)

### 3. Design System Architecture
- Choose architecture pattern (monolith, microservices, serverless)
- Define component boundaries and responsibilities
- Design data model and storage strategy
- Plan for scalability (horizontal, vertical, caching)

### 4. Select Technology Stack
- Backend framework (Django, FastAPI, Express, Spring)
- Frontend framework (React, Vue, Angular)
- Database (PostgreSQL, MongoDB, DynamoDB)
- Infrastructure (AWS, GCP, Azure, Kubernetes)
- Consider team expertise vs learning curve

### 5. Plan Implementation Approach
- Break into phases (MVP, Phase 2, Phase 3)
- Define deliverables for each phase
- Estimate timelines and resource needs
- Identify risks and mitigation strategies

### 6. Validate Strategy
- Review with stakeholders
- Prototype critical/risky components
- Validate assumptions with spike tasks
- Get approval before full implementation
