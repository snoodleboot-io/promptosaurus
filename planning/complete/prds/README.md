# Product Requirements Documents - Complete

This directory contains PRDs from finished projects.

## Purpose

Historical record of:
- What projects were delivered
- What the original vision was
- What was in/out of scope
- What success metrics were defined

## Usage

**READ-ONLY**: Reference and historical context.

Use these PRDs to:
- Understand delivered projects
- See how requirements evolved
- Compare planned vs actual scope
- Learn from past projects
- Onboard new team members

## Organization

Organize by release, phase, or keep flat:
```
complete/prds/
├── v1.0.0/
│   └── agent_framework.prd.md
└── v2.0.0/
    └── workflow_system.prd.md
```
