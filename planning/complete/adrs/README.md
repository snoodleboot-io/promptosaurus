# Architecture Decision Records - Complete

This directory contains ADRs from finished work.

## Purpose

Historical record of architectural decisions made in completed phases.
These ADRs document why the system is designed the way it is.

## Usage

**READ-ONLY**: For reference and learning, not active planning.

Use these ADRs to:
- Understand why past decisions were made
- Onboard new team members
- Avoid re-debating settled decisions
- Learn from past trade-offs

## Organization

Consider organizing by phase or feature:
```
complete/adrs/
├── phase1/
│   └── 20260301_database_choice.adr.md
├── phase2/
│   └── 20260315_execution_model.adr.md
└── v1.0.0/
    └── 20260401_authentication.adr.md
```

Or keep flat with clear naming and dates.
