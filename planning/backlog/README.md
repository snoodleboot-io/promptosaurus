# Planning Backlog

This directory contains future work that is not currently in scope.

## Purpose

Capture ideas, features, and architectural explorations for potential future implementation.

## Contents

- **adrs/** - Exploratory architecture decisions (not yet approved)
- **execution-plans/** - Rough execution plans for future phases
- **features/** - Feature ideas and specifications (not prioritized)
- **prds/** - Future project requirements (not scoped)

## Workflow

1. **Ideation**: Capture feature ideas and requirements
2. **Exploration**: Document potential approaches in ADRs
3. **Rough Planning**: Create high-level execution plans
4. **Prioritization**: When prioritized, move to `../current/`

## Guidelines

- Don't overinvest in detailed planning here
- Keep specs rough and exploratory
- Move to `current/` when work is approved and scheduled
- Delete items that are no longer relevant

## Status Indicators

Consider prefixing files to indicate status:
- `DRAFT_` - Early idea, needs refinement
- `PROPOSED_` - Well-defined, awaiting approval
- `DEFERRED_` - Was in current but deprioritized
