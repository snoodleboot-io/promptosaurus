# Product Requirements Documents - Backlog

This directory contains PRDs for potential future projects.

## Purpose

Capture project ideas and requirements not yet in scope:
- **Vision**: Long-term product direction
- **Rough Requirements**: High-level needs
- **Strategic Options**: Potential directions
- **Deferred Work**: Projects postponed for later

## Status

**NOT SCHEDULED**: Projects here are not committed.

PRDs here are:
- Exploratory and subject to change
- Useful for strategic planning
- May be significantly revised when prioritized
- May be abandoned if strategy shifts

## File Naming Convention

Consider status prefixes:
```
DRAFT_{project_name}.prd.md
PROPOSED_{project_name}.prd.md
DEFERRED_{project_name}.prd.md
```

Examples:
- `DRAFT_v3_architecture.prd.md`
- `PROPOSED_enterprise_features.prd.md`
- `DEFERRED_mobile_app.prd.md`

## Lifecycle

1. **Creation**: Project idea captured
2. **Refinement**: Develop requirements as understanding grows
3. **Prioritization**: Move to `../../current/prds/` when approved
4. **Deletion**: Remove if no longer strategic

## Guidelines

- Keep PRDs high-level
- Focus on vision and value
- Don't overinvest in detailed requirements
- Update when strategy changes
