# Feature Specifications - Backlog

This directory contains specifications for potential future features.

## Purpose

Capture feature ideas not currently in scope:
- **Ideas**: Early-stage feature concepts
- **User Needs**: Identified problems to solve later
- **Rough Specs**: High-level requirements without details
- **Parking Lot**: Good ideas for future consideration

## Status

**NOT PRIORITIZED**: Features here are not scheduled.

Features here are:
- Ideas awaiting prioritization
- May be refined when prioritized
- May be abandoned if deprioritized
- Useful for roadmap discussions

## File Naming Convention

Consider status prefixes:
```
DRAFT_{feature_name}.feature.md
PROPOSED_{feature_name}.feature.md
```

Examples:
- `DRAFT_plugin_system.feature.md`
- `PROPOSED_graphql_api.feature.md`

## Lifecycle

1. **Creation**: Feature idea captured
2. **Refinement**: Flesh out details as understanding grows
3. **Prioritization**: Move to `../../current/features/` when scheduled
4. **Deletion**: Remove if no longer relevant

## Guidelines

- Keep specs rough and high-level
- Focus on problem and user value
- Don't overinvest in detailed requirements
- Update when new information emerges
