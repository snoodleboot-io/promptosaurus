# Execution Plans - Backlog

This directory contains rough execution plans for future work.

## Purpose

High-level planning for features not yet in scope:
- **Rough Estimates**: Ballpark effort and timeline
- **Task Sketches**: General approach without details
- **Dependency Mapping**: What needs to happen first
- **Risk Identification**: Potential blockers

## Status

**ROUGH PLANNING**: Not detailed or committed.

Plans here are:
- Exploratory and subject to change
- Useful for roadmap planning
- Not ready for implementation
- May be significantly revised when prioritized

## Guidelines

- Keep plans high-level
- Focus on approach, not detailed tasks
- Identify major dependencies and risks
- Don't overinvest in planning

## Lifecycle

1. **Creation**: Rough plan for future phase/feature
2. **Refinement**: Update when more information emerges
3. **Promotion**: Move to `../../current/execution-plans/` when scheduled
4. **Deletion**: Remove if no longer relevant
